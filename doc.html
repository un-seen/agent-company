Overview

Agent Company is a modular framework for building autonomous agents. It is designed to help developers build sophisticated multi-agent systems. The framework leverages Redis for messaging and provides a flexible API to incorporate custom tools and agents.

Key Features:

  • Modular architecture for building autonomous agents
  • Flexible API for custom tool integration
  • Redis-based messaging system
  • Support for various environment types
  • Multi-agent system capabilities

Installation

You can install Agent Company using pip:

pip install agent-company

Requirements

  • Python 3.8+
  • Redis (for messaging)

Clone from GitHub

Alternatively, you can clone the repository directly:

git clone https://github.com/un-seen/agent-company.git
cd agent-company
pip install -e .

Core Components

Agent Company consists of several key components that work together to create a flexible agent system:

Agent

The autonomous entity that interacts with the environment, makes decisions, and takes actions based on its goals and perceptions.

Driver

Manages the execution flow and lifecycle of agents, handling initialization, scheduling, and termination.

Environment

The context in which agents operate, providing the state information and accepting agent actions.

Protocol

Defines the communication standards between components, ensuring proper message formatting and routing.

Component Relationships

These components interact in a structured way to create a complete agent system:

  • Agents observe the environment and take actions within it
  • The Environment provides observations to agents and processes their actions
  • The Driver manages the execution flow, determining when agents act and updating the environment
  • The Protocol ensures all components can communicate effectively

Features

Modular Architecture

Build and combine agent components independently, allowing for flexible system design and easy maintenance.

Redis Messaging

Leverages Redis for reliable and scalable messaging between agents and components.

Flexible API

Incorporate custom tools and agents with a straightforward API designed for extensibility.

Environment Support

Comprehensive support for various environment types with well-defined interfaces.

Environment Types

The Agent Company framework supports a variety of environment types that determine how agents interact with their surroundings. The following table organizes the environment types, providing a structured view for clarity:

Category Subcategory Description Example
Observability Fully Observable The agent has complete information about the state of the environment at any time, requiring no history tracking. Chess, where the board and moves are fully visible.
Partially Observable The agent has incomplete information, needing to infer hidden states. Driving, where road conditions beyond corners are unknown.
Determinism Deterministic The next state is completely determined by the current state and action, with no randomness. Chess, where each move has a definite outcome.
Stochastic The next state involves randomness, not fully predictable by the agent. Self-driving cars, affected by unpredictable driver behaviors.
Interaction Type Competitive Agents compete to optimize their own objectives, often in zero-sum games. Chess, where players aim to defeat each other.
Cooperative Agents work together to achieve a common goal, coordinating actions. Multiple self-driving cars cooperating to avoid collisions.
Number of Agents Single-agent Only one agent operates in the environment, focusing on its task. A person navigating a maze alone.
Multi-agent Multiple agents interact, potentially competing or cooperating. A game of football with 11 players per team.
Environment Dynamics Static The environment remains unchanged over time, except by agent actions. An empty house, where entering doesn't alter surroundings.
Dynamic The environment changes independently of agent actions over time. A roller coaster ride, constantly in motion.
State and Action Space Discrete The environment has a finite number of states or actions, countable and distinct. Chess, with a finite set of possible moves per game.
Continuous The environment has an infinite number of states or actions, not countable. Self-driving cars, with continuous driving and parking actions.
Task Structure Episodic The agent's actions are divided into independent episodes, each with no dependency on previous ones. A pick-and-place robot inspecting parts on a conveyor belt, deciding per part.
Sequential The agent's actions are interdependent, with previous decisions affecting future ones. Checkers, where each move impacts subsequent strategies.
Environment Knowledge Known The agent knows the complete model, including how actions lead to state transitions. Chess, with known rules and outcomes for moves.
Unknown The agent does not know the model, requiring learning through interaction. An agent in an unexplored environment, needing to discover dynamics.

Understanding these environment characteristics helps developers design appropriate agent architectures and strategies for their specific use cases.

Usage Examples

Basic Agent Setup

Here's how to set up a basic agent with Agent Company:

from agent_company import Agent, Environment, Driver

# Create a simple environment
env = Environment(type="fully_observable")

# Create an agent
agent = Agent(name="my_agent")

# Create a driver to manage the agent and environment
driver = Driver(agents=[agent], environment=env)

# Run the system
driver.run()

Multi-Agent System

Setting up a multi-agent system with Redis messaging:

from agent_company import Agent, Environment, Driver, RedisMessaging

# Initialize Redis messaging
messaging = RedisMessaging(host="localhost", port=6379)

# Create environment
env = Environment(type="partially_observable", messaging=messaging)

# Create multiple agents
agent1 = Agent(name="agent1", messaging=messaging)
agent2 = Agent(name="agent2", messaging=messaging)

# Create driver
driver = Driver(
    agents=[agent1, agent2],
    environment=env,
    messaging=messaging
)

# Run the system
driver.run()

API Reference

Agent

The primary class for creating autonomous agents.

class Agent:
    def __init__(self, name, messaging=None, config=None):
        """
        Initialize an agent.
        
        Args:
            name (str): Unique name for the agent
            messaging: Communication system for the agent
            config (dict): Configuration parameters
        """
        
    def observe(self, environment):
        """
        Get an observation from the environment.
        
        Args:
            environment: The environment to observe
            
        Returns:
            Observation data
        """
        
    def decide(self, observation):
        """
        Make a decision based on the observation.
        
        Args:
            observation: Data observed from environment
            
        Returns:
            action: The chosen action
        """
        
    def act(self, action, environment):
        """
        Perform an action in the environment.
        
        Args:
            action: The action to perform
            environment: The environment to act in
            
        Returns:
            result: Result of the action
        """

Environment

Base class for creating agent environments.

class Environment:
    def __init__(self, type="fully_observable", config=None):
        """
        Initialize an environment.
        
        Args:
            type (str): Type of environment (fully_observable, partially_observable, etc.)
            config (dict): Configuration parameters
        """
        
    def get_observation(self, agent_id):
        """
        Provide an observation to an agent.
        
        Args:
            agent_id: Identifier for the requesting agent
            
        Returns:
            observation: Data about the environment state
        """
        
    def process_action(self, agent_id, action):
        """
        Process an agent's action and update environment state.
        
        Args:
            agent_id: Identifier for the acting agent
            action: The action being performed
            
        Returns:
            reward: Numerical feedback about the action
        """
        
    def reset(self):
        """
        Reset the environment to an initial state.
        
        Returns:
            initial_state: The environment's initial state
        """

Driver

Manages the execution flow of the agent system.

class Driver:
    def __init__(self, agents, environment, config=None):
        """
        Initialize a driver.
        
        Args:
            agents (list): List of agents to manage
            environment: The environment agents operate in
            config (dict): Configuration parameters
        """
        
    def run(self, episodes=1):
        """
        Run the agent system.
        
        Args:
            episodes (int): Number of episodes to run
            
        Returns:
            results: Performance metrics and outcomes
        """
        
    def step(self):
        """
        Execute a single step of the agent system.
        
        Returns:
            state: Current state after the step
        """
